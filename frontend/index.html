AURA Security Operations Center (v2.0) Data Handling Notice

This application is a local Security Operations Center (SOC) agent designed to assess individual endpoint vulnerability postures. To evaluate threats, retrieve real-time analytics, and offer mitigation plans, the agent requires authorized system access to monitor configurations.

1. Monitored Data Points
  • Active Network Sockets: Identifies listening ports, process owners, and connection details to map malicious inbound or outbound activity.
  • System Registry Access: Reads installed software repositories and historical USB storages (via standard Registry parameters) to audit vulnerabilities.
  • Account Password Policies: Runs non-elevated queries on local account rules to verify compliance with recommended password policies.
  • Local File Scanning: Scans target directories (Desktop & Documents) exclusively for plaintext file extensions matching insecure password patterns to help lock credentials.
2. Ethical Privacy & Data Storage Standards

Strict Local Operations: All telemetry audits, scans, and system checks operate 100% locally. The application does not send files, registry keys, network packages, or personal user information to remote servers. All telemetry remains stored on the local encrypted SQLite client database (`soc_dashboard.db`).

3. Legitimate Compliance

By granting permissions below, you allow the AURA SOC system to run non-intrusive local scans. No rootkits, driver modifications, or persistent background service processes are created. Scans can be disabled or paused at any point.
